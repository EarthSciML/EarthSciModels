{{- /* Markdown image render hook. Hugo does NOT prefix the site's baseURL onto root-relative ("/foo") image destinations in markdown content. When the site is published under a path prefix (e.g. https://earthsciml.github.io/EarthSciModels/), an emitted ![](/plots/x.png) renders as , which the browser resolves at the apex (earthsciml.github.io/plots/x.png → 404) instead of inside the project prefix (earthsciml.github.io/EarthSciModels/plots/x.png). Generated component pages (tools/esm_to_docs.py) emit root-relative plot refs by design — the asset is copied to docs/static/plots//. Strip the leading "/" and run the rest through relURL so the baseURL path is prepended; protocol-relative ("//host/x"), absolute ("https://..."), and page-relative ("foo.png") destinations are passed through untouched. Note: Hugo's relURL/absURL only honor the baseURL's PATH component when the input is a *relative* path (no leading "/"). A leading-"/" input is treated as already-rooted and returned as-is by relURL — which is why the raw markdown ![](/plots/x) emits src="/plots/x" and 404s under the /EarthSciModels/ project prefix. Trimming the slash before piping is what actually applies the prefix. */ -}} {{- $dest := .Destination -}} {{- if and (hasPrefix $dest "/") (not (hasPrefix $dest "//")) -}} {{- $dest = (strings.TrimPrefix "/" $dest) | relURL -}} {{- end -}}
